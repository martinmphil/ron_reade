| Term                       | Definition                                                                                                                                                                                                   |
|----------------------------|--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| Artificial Neural Network  | The machine learning model (specifically for Text-to-Speech) that runs locally in the user's browser to convert text strings into synthesised audio.                                                         |
| Audio Buffer               | A temporary digital storage area in the browser's memory that holds processed audio data before it is sent to the speakers. When flushed, all current data is purged and playback is terminated immediately. |
| Chunk                      | A segment of the input text. Large bodies of text are broken into smaller pieces to allow the neural network to process manageable chunks.                                                                   |
| Chunking                   | The act of segmenting text into a chunk. With respect to UI updates the text chunking process is considered to be instantaneous.                                                                             |
| Flush Audio                | Terminate current playback; Reset audio player seek-bar to 00:00; Remove all audio data.                                                                                                                     |
| Halt Processing            | An abort signal from the user that immediately terminates the neural network's processing, stops all computation, and flushes any partially generated audio data                                             |
| Hierarchical Text Chunking | Segment the Text Manifest into chunks at word boundaries, respecting paragraph and sentence limits.                                                                                                          |
| Model Acquisition          | Retrieving cached model files or downloading model from the internet with timely retry strategy                                                                                                              |
| Parallel Regions           | Distinct logical areas of the app (Input vs. Output) that operate simultaneously. Actions in one region (like typing) do not necessarily block actions in the other (like audio playback).                   |
| Text Manifest              | Immutable snapshot capturing current text-area content as an input source for the text-to-speech process.                                                                                                    |
| Valid Text                 | A series of text validation steps described in the Requirements Document                                                                                                                                     |
